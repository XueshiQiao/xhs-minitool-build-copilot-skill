# 产品方向：小红书想要什么样的小工具

> 技术可行 ≠ 值得做。这份文档判的是后者。

---

## 一、先读懂那些限制背后的意图

把禁用清单排在一起看，会发现它们不是随手砍的：

```
禁文件下载  ·  禁剪贴板  ·  禁 target="_blank"  ·  禁跳转站外
禁 window.open  ·  禁小工具互跳  ·  禁 iframe
```

**这不是技术限制，这是产品设计。** 每一条都在做同一件事：
**不让价值和流量流出笔记**，把用户和互动牢牢摁在笔记页与评论区里。

再叠加另一组信号：

```
完全离线  ·  纯本地  ·  无网络  ·  禁 WASM  ·  禁 Worker
```

官方文档自己写了「需要 AI 图像处理等重计算的场景本期无法支持」——
说明他们要的是**轻量、自包含、瞬时的互动件**，不是重型 SaaS。

再加上官方把这件事叫「vibecoding」并面向**创作者**内测（不是面向专业开发），
意图就很完整了：

> 让内容创作者随手做出**配合自己选题**的小互动，
> 读者点开即用、用完回到评论区，整个闭环不出笔记。

**逆着这个意图设计的工具，即使能跑起来也没人用。**

---

## 二、三条契合度标准

用户的点子过这三条，**不过关就要改**：

### 标准 1 · 读者零素材就能玩

读者是刷到笔记顺手点开的，手里**什么都没准备**。

- ✅ 点开就能输入 / 选择 / 开始
- ❌ 需要先找两张对比图、先准备一份数据、先拍好照片

> 这条最容易被忽略。很多点子技术上完全可行，
> 但它的真实用户其实是**创作者自己做图**，而不是笔记的读者——
> 偏偏容器又禁了导出，连创作者那条路也堵死了，**两头不靠**。

### 标准 2 · 结果因人而异，值得截图

小工具的传播靠**读者截图发出去**（因为这是唯一的出口）。

- ✅ 输入自己的信息 → 得到专属于自己的结果 → 想晒
- ❌ 所有人点开看到的都一样（那不如直接写在笔记正文里）

判断方法：**这个结果值得他截图发出去吗？值得在评论区报数吗？**

### 标准 3 · 纯本地、算得快、结果留在笔记内

- ✅ 前端一算就出，没有等待
- ❌ 需要联网、需要模型、需要几秒以上的计算

## 三、契合的形态

按契合度从高到低：

| 类型 | 例子 | 为什么契合 |
|---|---|---|
| **测算 / 测试类** | 性格测试、风格测试、匹配度、各类量表 | 三条全中，结果天然因人而异且值得晒 |
| **计算器类** | 房贷、卡路里、装修预算、囤货省钱、单位换算 | 零素材、结果个性化、纯本地 |
| **抽签 / 决策类** | 今天吃什么、穿搭抽签、随机挑战 | 极轻，互动成本最低，容易反复玩 |
| **轻互动小游戏** | 点击、记忆、反应力 | 涨互动，读者愿意在评论区报成绩 |
| **对照 / 查询表** | 尺码对照、色号对照、成分速查（数据打包成 json） | 零素材、即查即走，实用性强 |
| **打卡 / 收集类** | 习惯打卡、盖章收集 | 结果随时间积累，越用越值得晒（注意数据持久化的黄线） |

---

## 四、不契合的形态（以及怎么改）

| 想做的 | 问题 | 改造方向 |
|---|---|---|
| 图片对比 / 修图 / 拼图 | 读者没素材；且导出被禁，创作者也带不走成品 | 改成「上传一张图 → 得到因人而异的分析结果」：主色卡提取、个人色彩季型、五官比例 |
| 文档 / 表格处理 | 选文件只能选图片和视频，根本传不进来 | 无解，劝退 |
| 需要账号 / 排行榜 / 多人协作 | 无网络，数据互相隔离 | 改成纯单机的自我对比（和上周的自己比） |
| 长期记账 / 健康档案 | 数据不保证持久化且无备份 | 缩短时间尺度：做「本月」而不是「历年」，主打阶段性截图 |
| AI 生成 / 识别类 | 不能联网也不能跑 WASM | 换成规则化的伪个性化（输入 → 查表 → 出结果），照样能做出千人千面的感觉 |

## 五、一个真实的反例

> 有人想把一个成熟的网页版「图片前后对比」工具搬进小工具。

**技术判定**：完全可行。Canvas 2D 完整可用、`<input type="file">` 能选图、
纯本地处理天然不需要网络。看起来是完美命中。

**产品判定**：三条标准只过一条。

1. 读者零素材？❌ 必须先有两张对比图
2. 结果因人而异？❌ 结果就是他自己传的图
3. 纯本地算得快？✅

更致命的是：**这个工具的核心功能「下载对比图」在容器里根本做不出来**——
`a[download]`、blob 下载、剪贴板、长按菜单，四条出口全被禁。
用户唯一能拿走结果的方式是自己截屏。

**结论**：技术上一路绿灯，产品上两头不靠。
这就是为什么**可行性判定必须包含产品判定**，只判技术会把人带进沟里。
